Phase Test 2

In this phase test, you will complete four functions inside a simple turn-based RPG combat program. The player fights a randomly selected enemy until one side's HP reaches zero. Enemy data is stored across three parallel arrays: one for names, one for HP values, and one for attack values. A single random index selects the same enemy across all three arrays. 

A partial implementation is provided. You must complete the functions.

Look for comments "// START OF YOUR CODE" and "// END OF YOUR CODE". These show you where to write the solution code. Do not add or change any code that is outside these comments.

Download, extract and open the rpg_simulation project in Visual Studio.

Task 1

Complete selectEnemy(). It receives three output pointer parameters: a string*, an int*, and an int*. The function must:

  • Generate a random index in the range [0, ENEMY_COUNT] using rand() and the sizeof idiom or ENEMY_COUNT directly
  • Use the index to read from each of the three parallel arrays
  • Write the values into the output parameters by dereferencing each pointer

Task 2

Complete isDefeated(). It receives a single int* pointing to an enemy's current HP. Return true if the value it points to is zero or below, false otherwise. You must dereference the pointer to read the value. Do not use a local copy passed by value.

Task 3

Complete applyDamage(). It receives the enemy's name, a pointer to their current HP, the damage amount, and a char* buffer for logging. The function must:

  • Subtract damage from the value pointed to by hpPtr
  • Ensure the HP cannot fall below zero

Example: Orc takes 7 damage. HP remaining: 43 logBuffer is declared in playEncounter() and passed in. Do not declare a new array inside this function.

Task 4

Complete main(). Call playEncounter() to run one combat encounter with a random enemy.

Important Notes

  • Your submission must be a single .cpp file (i.e. main.cpp). Do not split into header/source files.
  • Partial marks are available. A function that compiles and partially works will receive more credit than one left blank.
  • Include more code comments to clearly demonstrate your understanding of the purpose, functionality, and logic behind each line of code

Submission

  1. One file containing C++ source code (i.e. main.cpp)
